Quick Installation

Get started with Super Agent CLI in minutes.

bash 
bun add -g @involvex/super-agent-cli
bash 
npm install -g @involvex/super-agent-cli

Prerequisite: Node.js 18+ and Bun or npm must be installed on your system.

Usage

Interactive and headless modes for any workflow.

Interactive Mode

Start a conversational AI assistant in your terminal. 

super-agent
# or with specific directory
super-agent -d /path/to/project

Headless Mode

Process a single prompt and exit for scripting and automation. 

super-agent --prompt "show me the package.json"
super-agent -p "create a new file called example.js"
super-agent --prompt "run tests" --directory /path/to/project

Command Line Options

Option Description
-V, --version Output the version number
-d, --directory /path/to/project Set working directory
-u, --base-url https://api.openai.com/v1 API base URL
-m, --model gpt-4o AI model to use
-p, --prompt "prompt" Process a single prompt and exit (headless mode)
--max-tool-rounds "rounds" Maximum number of tool execution rounds (default: 400)
-h, --help Display help for command

Available Commands

Command Description
about Show information about the Super Agent CLI

Configuration

Super Agent CLI uses two types of configuration files.

User-Level Settings

~/.super-agent/settings.json

Global settings that apply across all projects. Stores API key, base URL, and default model. 

{
  "active_provider": "gemini",
  "providers": {
    "gemini": {
      "id": "gemini",
      "provider": "gemini",
      "model": "MODEL_NAME",
      "api_key": "API_KEY",
      "base_url": "BASE_URL (optional)",
      "default_model": "MODEL_NAME"
    }
  }
}

Project-Level Settings

.super-agent/settings.json

Project-specific settings in your current working directory. Includes MCP server configurations. 

{
  "active_provider": "gemini",
  "providers": {
    "gemini": {
      "id": "gemini",
      "provider": "gemini",
      "model": "MODEL_NAME",
      "api_key": "API_KEY"
    }
  },
  "mcpServers": {
    "linear": {
      "name": "linear",
      "transport": "stdio",
      "command": "npx",
      "args": ["@linear/mcp-server"]
    }
  }
}

Priority Order

  1. Project-level (`.super-agent/settings.json`) - takes highest priority
  2. User-level (`~/.super-agent/settings.json`) - fallback for global settings
  3. System default (`gemini-3-pro-preview`) - used when no other settings exist

MCP Tools

Extend capabilities with Model Context Protocol servers.

Managing MCP Servers

# Add an stdio-based MCP server
super-agent mcp add my-server --transport stdio --command "bun" --args server.js

# Add an HTTP-based MCP server
super-agent mcp add my-server --transport http --url "http://localhost:3000"

# Add with environment variables
super-agent mcp add my-server --transport stdio --command "python" --args "-m" "my_mcp_server" --env "API_KEY=your_key"

# Add from JSON configuration
super-agent mcp add-json my-server '{"command": "bun", "args": ["server.js"], "env": {"API_KEY": "your_key"}}'

# List all configured servers
super-agent mcp list

# Test server connection
super-agent mcp test server-name

# Remove a server
super-agent mcp remove server-name

Available Transport Types

stdio

Run MCP server as a subprocess (most common)

http

Connect to HTTP-based MCP server

sse

Connect via Server-Sent Events

Linear Integration Example

To add Linear MCP tools for project management:

# Add Linear MCP server
super-agent mcp add linear --transport sse --url "https://mcp.linear.app/sse"

This enables Linear tools like:

  • Create and manage Linear issues
  • Search and filter issues
  • Update issue status and assignees
  • Access team and project information

Tool Execution Control

Control tool execution rounds for different use cases:

# Limit tool rounds for faster execution on simple tasks
super-agent --max-tool-rounds 10 --prompt "show me the current directory"

# Increase limit for very complex tasks (use with caution)
super-agent --max-tool-rounds 1000 --prompt "comprehensive code refactoring"

# Works with all modes
super-agent --max-tool-rounds 20  # Interactive mode
super-agent git commit-and-push --max-tool-rounds 30  # Git commands

Use Cases

  • Fast responses: Lower limits (10-50) for simple queries
  • Complex automation: Higher limits (500+) for comprehensive tasks
  • Resource control: Prevent runaway executions in automated environments
